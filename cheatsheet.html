How to Create a new object:
var nate = {};

How to add properties to an object:
var nate = {
name: "nate",
age: 30,
"other fav food": "burgers"
}
If the Property is not a valid name, it must be quoted.

How to create constructor:
function Person(name,age){
this.name = name;
this.age = age;
}
How to create new ppl from our Person Constructor:
var nate = new Person("Nate", 30);
var nicholas = new Person("Nicholas", 40);

Methods are simply properties that hold function values.
How to create a method on the "nate" object:
nate.run = function(where, speed){
console.log("nate ran to the " + where + "at" + speed + "MPH!");
}
This method takes two parameters and prints out a phrase with those 2 parameters.

How to delete properties:
delete nate.age; //this will delete the age property.
delete nate; //this will delete the nate object.

Add a new method to a prototype:
Rabbit.prototype.dance = function() {
console.log(“The “ + this.type + “ rabbit dances a jig.”);
};
invocation:
instanceName.dance();

Overriding derived properties enables us to express exceptional
properties while letting non-exceptional or default objects imply
take a standard value from their prototype.
Rabbit.prototype.teeth = “small”;
console.log(killerRabbit.teeth); // => small
killerRabbit.teeth = “long, sharp, and bloody”;
console.log(killerRabbit.teeth); // => long, sharp, and bloody
console.log(Rabbit.prototype.teeth); // => small

W used an object as a way to associate values with names by creating
properties for the names and giving them the corresponding value as their value. Example:
var map = {};
function storePhi(event, phi) {
map[event] = phi;
}
storePhi(“pizza”, 0.069);
storePhi(“touched tree”, -0.081);
Using this function, we could for/in loop and test whether a name is in the map object using the in operator, but the object’s prototype gets in the way.

To define a new property:
Object.defineProperty(Object.prototype, “hiddenNonsense”,
{enumerable: false, value: “hi”});

console.log(map.hasOwnProperty(“toString”)); // => false
Use the .hasOwnProperty(“toString”)); to handle the regular in
operator in a for/in loop as it claims that the Object.prototype properties exist in our object.
