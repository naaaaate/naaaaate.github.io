Function Declarations vs Expressions (for Beginners)

Jul 22nd 2015

Today, I wanted to discuss Function Declarations and Function Expressions. (I know what you're thinking.."Function whaaa?!?!")... Stay with me..

As I mentioned in a previous blog, Functions are very important in JavaScript! They create their own scope and are considered "First Class Citizens!" But lets take a deeper dive into what a function declaration is vs a Function Expression.

Simply put, a function declaration is when you declare a function and it is the first word in the statement. Think back to when we talked about declaring variables and functions and compiling them with the "Phase 1 JavaScript Compiler". Here's an example: 


                function bar() {
                  var foo = “magic”;
               }

This is a function declaration bc the keyword "function" is the first word in the statement. Now, lets compare that with a function expression: where the function keyword is not the first word in the statement: 


                var foo = function bar() {
                  var foo = “baz”;

                  function baz(foo) {
                    foo = bar;
                    foo;
                  }
                  baz();
                };

Here we can see that in the first line, the keyword "function" is not the first word. It's actually being assigned to the var foo. Therefore this is a function expression.. More specifically though, its called a "Named Function Expression" bc after the keyword "function" there is a name: "bar". Sometimes you can have what's called an "Anonymous Function Expression" where a function expression is declared with no "name" after the keyword "function".

What's great about function expressions is that they can refer to themselves within themselves. (kind of how some people refer to themselves in the third person... which is kind of aggravating.) So within the function expression on the first line called "bar" , we can now refer to this function expression as we do here.. 


                //var foo = function bar() {
                  //var foo = “baz”;


                  //function baz(foo) {

                    foo = bar;  -- here is where we refer to function expression bar.

                    //foo;
                  //}
                  //baz();
                //};
                foo();
                bar(); //Error!

In fact, that is one of the defining characteristics of a function expression: that function expressions can refer to themselves within themselves and are not accessible outside of the function. So if you try to access the function expression bar by calling bar() in the global scope, you will get an error!

Now as I mentioned earlier, we could have an "Anonymous function expression" but it is better to use "Named Function Expressions" for 3 reasons:

  1. If theres no: "named expression" above (ie: function bar) you cannot refer to itself from inside of itself.
  2. If there are errors in your code, then during the debugging phase, an anonymous function does you no good in trying to find the exact function where the issue occurs if there are multiple functions called: "anonymous".
  3. A Named Function Expression will self document its code, by giving itself a name that helps you understand what its doing. So if we called it Handler, you would know that the function is handling some kind of event.

So to recap: Function Expressions can refer to themselves insid of themselves (ie: people who refer to themselves in third person.. ie: "Smeegle"). Function Expressions can be anonymous or named, and finally: Named Function Expressions are better to use then Anonymous Function Expression. I really hope that helps in your understanding of Function Declarations vs Function Expressions.
