Ruby Classes vs Javascript Constructor Functions

A brief tutorial on Ruby Classes & JavaScript Constructors

4/12/15

This week I want to take a look at the slight differences between Javascript Constructor Functions and Ruby Classes. You may or may not already know that JavaScript does not contain any classes. Furthermore, in case you didn't know, classes are a great way to group similar things. So when you hear classes, think of a grouping of things like humans, animals, tools, etc.. Let's take a look at some of the key differences between Ruby Classes and Javascript constructors.


In Ruby we created classes as a way to set a blueprint for creating objects of that class. Put another way: we made a factory that builds objects of whatever that factory is. So if our factory is a car factory, the object that would come out of our factory is a car object. Now, in Javascript, we do not have the luxury of just creating a class factory, if you will... Instead, we have to actually create what's called a Function Constructor, that will create an object with properties that we input. This is very similiar to creating a Class and inside of its initialize method we set variables equal to whatever arguments we pass into the class at the time we create the new class.
So in JavaScript, let's setup a function constructor that will create many objects of this from. Let's take a look at how we would create an athlete function constructor that creates atheletes with different names and properties.. 

              function Athlete(name, age, sport){
                this.name = name;
                this.age = age;
                this.sport = sport;
              }

              var nate = new Athlete("Nathan", 30, "Basketball");
As you can see in this example I have created a function constructor called Athlete. It takes in the parameters, name, age, and sport. At the bottom when we initialize or create a new athelete object called: nate, we do so with the new keyword, a call to Athlete constructor, and we pass in the arguments that match the parameters we set.
So, now there is an object called nate that has certain properties we can access by calling using the dot notation and the name of the property. For example.. 
              console.log(nate.name + "is" + nate.age + "years old and likes to play: " + nate.sport);
This simple console.log statement will print out to the console: "Nathan is 30 years old and likes to play: Basketball.""
Now you can kind of see the importance and benefits of creating a constructor function this way, because it allows us to create multiple athletes with different names, age, and sports that they play and we can use them in a program and access them simply by calling their object name, dot, their property name.


That touches on some of the basic differences between JavaScript Constructor Functions and Ruby Classes. Yes they are similar in a way, but overall they differ in the way they create objects in their syntax. Ruby classes pass in new attributes with the initialize method and calls them with attribute accessors, readers and writers, and Javascript passes in new properties with the Function Constructor creation and calls them with the object.property syntax. I hope this tutorial has helped you to better understand the main difference between Ruby Classes and Javascript Constructor Functions. Thanks for reading!
