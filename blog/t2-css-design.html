Inline vs Inline-block.

Understanding the difference in HTML Display Properties.

3/8/15

Today I want to clear up the differences between an inline block element and an inline element. For starters if you’re new to programming in html and css please note that you have elements that you use to design your page. Elements can be (div, p, h1, a, etc..) tags. Now display properties such as: inline, inline-block and block will make these elements show up on the webpage in different ways. Today I’m going to show you a few examples of what these differences are, how each element is affected when you set it to the different properties and how it affects the page and everything around it.

Lets look at inline elements first. Inline elements do exactly as their name says: they stay in the same line as its surrounding text, paragraphs, pictures etc.. Whatever is next to it, will be on the same line as this inline element. Another way of saying it is: the inline-element stays in the same left to right "flow" of the document, from line to line of the doc.

Now a few things to note about inline elements are: 1. when u declare an element as inline, it will NOT start a new line after it. Everything just remains the same. Some elements like block elements always start a new line after it as Block elements don’t like other elements and objects on the same line as itself, but not inline elements.. with inline elements, everything will still be on the same line. 2. You cannot set its height and width (the inline element will only be as big as the content inside of it). Also, when it comes to margins think left and right… only the left and right margins expand when u set them, the top and bottom margins stay the same. 3. Finally, when it comes to padding, i’ve found that: only the bottom, left and right padding inside the element will expand or decrease. The top is not affected.

Lets look at block elements: For block elements, think of Legos you can stack on top of one another. Block elements will stack on top of each other meaning they will not allow other elements to be by their side. You can define certain height and width for blocks, however even if you set a width to a block element, it still will not allow any other elements to be by its side. They like their own space which means other elements will have to get their own line in the html doc. When you set its margin and padding, all four sides (top, bottom, left and right) will either expand or decrease. One more thing to note is blocks automatically take the entire width of the screen if u don’t set an actual width. Look at the before and after screenshots of blocks..

Note how the margins will create space between the other elements above and below it (remember nothing is allowed on its side) where as inline elements only created space bt elements on its own line (the left and right of it). Also take note of the padding of a block.. the top, bottom, left, and right padding inside will expand or decrease if u set its properties.

Finally, lets take a look at inline-blocks:
Rememeber with inline elements: the height and width are not affected when u set it, only the space on the left and right of it will move when u set them (aka: Margins), and inside of it (aka: Padding), only the left, right, and bottom will expand or decrease if you set its properties. Inline-block elements have characteristics of both blocks and inline elements.
Lets look at some examples:

They still do not start new lines. Everything stays in the same flow of the document and elements and text and pictures will remain on the left and right of it like a happy family. Also, just like blocks, their height and width can be adjusted, as well as all sides of their margins and paddings. So the main difference between inline and inline blocks is that with inline-blocks:
the actual element itself will change, and at the same time allow other elements and objects to be on the same line as it. It does not make other elements get their own line. And with regular inline elements:
The element does not change its height or width, and only left and right margins and padding can be expanded or decreased if you set them. (Also bottom padding changes with inline elements.)
