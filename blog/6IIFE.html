IIFE Patterns in JavaScript (for Beginners)

Jul 25th 2015

IIFE, IIFE, IIFE... such a funny name, yet probably one of the most popular JavaScript patterns used by developers. What is an IIFE? What is a pattern? Why do I need to know about any of this..? So many great questions.. lets take a deep dive into IIFE's.

First of all, an IIFE is short for an: Immediately Invoked Function Expression. (Remember my blog on Function Expressions ?). All an IIFE is, is a Function Expression (opposite of a function declaration) that is immediately run (or fancier word: 'invoked') at Runtime. So why would we ever want to use an IIFE?

To answer this question, remember that when developers say IIFE, they are referring to a pattern in JavaScript. Think of patterns like cookie cutters. When you use the cookie cutter, you have perfect looking cookies in the exact design you wanted. Furthermore, you keep using the cookie cutter because you know it will produce the same great results every single time.

The same can be said of the IIFE pattern, you know that by using this pattern you will get the same great results of a well organized JavaScript program. Now, the main thing you need to know about an IIFE is that an IIFE will take your function expression and make everything inside of it private! (in JavaScript, you'll find that its not a good idea to expose functions and variables unnecessarily to the outside global scope). We'll look at some examples in a few, but if there's one point I want to get across to you that I wish someone would've told me, its exactly what I just mentioned: (in very basic terms) IIFE's make everything declared inside of it private! (remember the whole blog I wrote on the importance of scope? You do NOT want to make your functions and variables available for anyone to access. It's the same reason you don't just leave your car doors unlocked with the windows down, or your front door wide open for anyone to walk inside.

Now, Lets look at some code: 


                var foo = “foo”;

                (function() {
                  var foo = “foo2”;
                  console.log(foo); //“foo2"
                }) ();  //it immediately invokes itself here..

              console.log(foo); //“foo”

A few things to notice here:

  1. The IIFE Syntax: notice how the second line of code, there is a function expression and not a function declaration. Again, it is a function expression because it begins with a parenthesis "(" and not the keyword: "function".
  2. The parentheses "()" at the end of the function expression: This is what actually runs (or invokes immediately) the function expression at runtime. No need to call it from below your function like you may have seen in the past. (ie: nate() .. or foo() etc..
  3. Privacy: Remember one of the MAIN reasons to use the IIFE pattern is for privacy. When you think IIFE think privacy! What does that mean, well.. look at the line inside of the function expression IIFE that says: "console.log(foo)".. it returns "foo2" because its calling the foo above it inside the IIFE. Now, look at the last line that says: "console.log(foo)" .. That line will return "foo" because it only has access to the global scope var foo and not the IIFE scope var foo.
    4. Again, please remember that we use the IIFE pattern to make its scope (or the personal space inside the Function Expression) private! Now we are also immediately invoking the Function expression so that there is no need to call it and expose its name to the public. If what I just said was confusing, let me try another way.. if we had a regular function: 
    
              function nate() {
                  var foo = “foo2”;
                  console.log(foo); //“foo2"
                }
    to call this function we would have to type: 
    
              nate();
    Now if we had IIFE called nate it would look like this: 
    
              (function nate() {
                  var foo = “foo2”;
                  console.log(foo); //“foo2"
              })();
    Since an IIFE is immediately invoked at runtime, we do not have to manually type in nate(), because the IIFE is already invoked! Hope that helps clear things up about IIFE's.

So that is basically what an IIFE is and how it operates. When you think of an IIFE remember to think of 2 things: 1. Privacy and 2. Already unning. If you have any questions or comments about this blog, please hit me up on twitter (@n8leedbc)! thanks!
