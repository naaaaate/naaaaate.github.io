Arrays vs Hashes

A Brief Overview of Arrays and Hashes

3/14/15

This week we are going to talk a little bit about Arrays vs Hashes and will find out the similarities and differences between the two and when we would use one over the other.

Lets start by looking at Arrays:
If you are new to programming, please simply think of an array as a container that holds items inside of it. There are many types of containers that are there to hold anything you want: sugar, spices, soups for smaller containers, and clothes, shoes, tools for larger containers. The point here is that containers hold things. Arrays work the same way. Arrays can hold object of any type: strings, booleans, integers, floats, other arrays, and so on.. The great thing about arrays is that they order the objects that are entered into the container so that you can pull out any object if you know its index or number in that container. One thing to note though: An Array's index starts at Zero and not One. That basically means that to call the first element in your array, you would have to specify index zero (like this: a[0] - calls first element in array) and not like this: a[1].

Here are a couple ways to create a new array and add things to it:
1. Array.new Method: 

              a = Array.new
              a[0] = "first"
This will first create a new array. Second, it will add the string: "first" to the first element in the array. Remember: Array's have a zero index which means you start counting from 0 and not 1. So to refer to the first element in the array "a", you would say: a[0]. To refer to the second element in array "a", you would say: a[1] , and so on..

2. The Literal-Constructor array method: 

              a = []
              a= [1,2,'three', 4, true]
The first line creates a new array. The second line creates a new array and adds elements into it. This is a very convenient way to create an array.

Here are a few ways to retrieve and remove elements from an array:
1. .shift and .pop - will shift items to the left and remove the first element. 

              a = [1,2,3,4,5]
              a.shift(1)
              print a
              a.pop(2)
              print a
Here's what is happening:
  1. An array called a is set to: 1,2,3,4,5
  2. a.shift(1) will shift off to the left the first value.
  3. a is printed out: 2,3,4,5
  4. a.pop(2) will pop off the last 2, bc we told it pop 2 off the end.
  5. a prints out remaining element: 2,3

    6. 2. .unshift and .push - will add a new value to the end of your array: 
                  a = [1,2,3,4,5]
              a.unshift(0)
              print a
              a.push(6,7,8)
              print a
    Here's what is happening:
    1. An array called a is set to: 1,2,3,4,5
    2. a.unshift(0) will add 0 to the beginning of your array
    3. a is printed out: 0,1,2,3,4,5
    4. a.push(6,7,8) will add 6,7,8 to end of array
    5. a prints out: 0,1,2,3,4,5,6,7,8



    What are hashes then?
    Hashes are containers as well! They store objects but in what is known as a key value pair. Now if that's confusing, please simply think of a hash as a dictionary. Dictionaries have a key/value pair as well. When you look up a word (the Key), it shows you the definition right there (the Value). Now instead of having to know what an objects number is to return it in your code (ie: like an array), all you need to know is the name of the key and its value will be returned to you. This makes it a lot more convenient when trying to organize code and call certain specific values when the amount of keys begin to grow and grow. For instance, think of keys on your key ring.. if you had 2 or 3 keys, knowing which is which would not be hard, but if you had 100 or 1000 keys on your key ring, things might start getting a little confusing.

    Now, please note that: any key and any value can be any Ruby object. And as stated earlier, you can lookup any object value based on the name of its key. Also note that the correct way to assign a key to its value is to use the: "=>" operator.
    Lets look at some examples of creating hashes and adding key/value pairs to it.

    1. This is how to create a hash and add values to it: 

                  h = {}
              or
              h = Hash.new

              h["New York"] = "NY"
              h["Florida"] = "FL"
              h["Lousiana"] = "LA"
    Here's what is happening:
    1. A Hash called "h" is created w {} called literal constructor
    2. can also create with the Hash.new method call
    3. The key value: "New York" is set to abbreviation value: "NY"
    4. The key value: "Florida" is set to abbreviation value: "FL"
    5. The key value: "Louisiana" is set to abbreviation value: "LA"

    Finally, lets take a look at how to retrieve values from a hash:
    1. [] method or .fetch method:
    To retrieve the abbreviation NY from the "h" hash and assign it to a variable: 

                 NY_abbr = h["New York"]
             or
             NY_abbr = h.fetch("New York")
    Here's what is happening:
    1. We are using the bracket [] method which is prob most popular to retrieve the value stored in the "New York" key.
    2. or we can use the fetch method to get the value stored in the "New York" key.
    Please note: either method that use, you still need to know the key name to get its value.

    That sums up a breif introduction to hashes and arrays, and how to operate within them. I personally feel that you would use one over the other depending on the level of detail you would need for a list. For example: If you were creating a method that took in a number, and multiplied itself by every number below it till it got to 0, you would need to first subtract that number by 1 till you had all the numbers ordered in a list. But where would you store those numbers that you extracted? Thats right.. an array! No need for details of creating a key and setting it to a value for each number, because you want a quick and easy solution here. But if you were creating a movie type method that looked up a movie title by main actor, then you would want to use a hash because it is way more detailed and easier to return that value of the movie for you.

    I hope this tutorial on Hashes vs Arrays helped you to better understand what hashes and arrays are, what the differences between them are, how to use them, and what scenarios to use each in.
