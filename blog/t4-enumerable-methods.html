Enumerable Method: Map

A tutorial on how to use the Map Enumerable method

3/21/15

Today I am going to give a brief tutorial on the Enumerable method: Group_By. But before we begin let me briefly explain what an enumerable method is and how to use it.

Enumerable methods are methods that collections such as arrays and hashes can use to make their lives, as well as your life, easier. Let me explain this in the most basic terms that I can think of. When you think of the word 'Enumerable' think of the word: "Enumerator" which basically means numbered, As in you have the ability to step thru a collection (such as a hash or an array) one number or one element or one value of that collection at a time. Enumerable methods are therefore methods that you can use to step thru your collections one element or value at a time.


Now you may be thinking.. isn't that what a code block with the .each method does already? And you are absolutely right! that is what code block with the each method does.
ie: 

              a = [1,2,3,4,5]
              a.each {|a| puts a}
This code will print out every element in the array called: 'a'.
But when it comes to Enumerables, think of it as an addition to the .each method. It will do exactly as .each does and then some..
For example: Lets look at the Enumerable method called "Map".. 
              names = %w{David Chad Amy Ron Smith}
              names.map {|name| name.upcase}
            
              What this code does is:
              

              
    
                
  1.  Creates an array of string values as names. 
    2. 
                
  2.  Then map will return a new array of elements in names that are ran thru the block statement. 
    3. 
                
  3.  Here the block takes every name and Upcases it.
    4. 
                
  4.  Therefore, an array of those names is returned all uppercase.
    5.


So, just to recap a few of the highlights of using map..

  1. Map, like all other enumerables, works just like each, in fact it uses the each method to step thru each element in the collection.
  2. Map Always returns an Array. and the returned array is Always the same size as the original Enumerable.
  3. The elements returned will consist of whatever condition you set to each element inside of the code block. So you can upcase, downcase, sort it, return the length of each element, etc..
  4. Map will return a new object while each wil simply return its original collection.


I hope this tutorial on Enumerables and the Map method has helped you to better understand what Enumerables are and how to specifically use Map.
